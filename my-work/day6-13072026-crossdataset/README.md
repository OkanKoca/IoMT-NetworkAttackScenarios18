# Cross-dataset test set — the upstream study's own FlowMonitor outputs

A held-out **generalization test set** built from the published FlowMonitor XML
outputs of the upstream study
(`ramamr33/IoMT-NetworkAttackScenarios18`, Zenodo `10.5281/zenodo.16747386`,
git remote `upstream`).

The detector is *trained on our regenerated simulations*; this set lets us ask
how it behaves on the original study's **independent** data. It is a
domain-shift / robustness probe — the author's outputs are **never** mixed into
training.

## Build it

```bash
python3 make_author_testset.py        # pull XMLs from `upstream`, parse
# -> out/dataset.csv  (60 runs, same 13-feature schema as our own dataset)
```

Everything under `raw_author/`, `out/`, and `manifest_author.csv` is
regenerated by the script and git-ignored; only the script and this README are
tracked. Feature vectors are computed by the **same** parser as our own dataset
(`../day3-4-.../build_dataset.py`, called as a subprocess) so the feature
definitions cannot drift between train and test.

## What is in it

6 source classes × 10 seeds = 60 runs: `normal, dos, ddos, blackhole, mitm,
mqtt`. Labels come from the source folder. `label_binary` = `normal` vs
`attack`.

## Read before using the numbers

This set exposes real limits of a naive cross-dataset comparison; they are
findings, not defects to hide:

1. **Total covariate shift.** The author's topology differs from ours on every
   feature (9 flows/run vs 2-6; ~2.6× throughput; different port layout). A
   detector trained on our runs operates far outside its training range here —
   expect it to misread even the author's *normal* runs.
2. **Degenerate source labels.** In feature space the author's
   `dos`, `ddos`, and `blackhole` runs are **identical** to each other, and
   `mqtt` is **identical** to `normal`. So the six labels collapse to three
   distinct signatures. No classifier can separate labels that share a feature
   vector.
3. **Weak attack signal.** The author's attack runs differ from baseline by
   ~1% in delivery ratio / loss — inside normal run-to-run variance.
4. **Missing roles.** The author network has no `relay_in` (7070) or
   `telemetry` (9090) ports, so `telemetry_throughput_mbps = 0` throughout and
   `delivery_ratio` is pump-path only. `greyhole` (our novel attack) has no
   counterpart in the author data.

Bottom line: use this set to demonstrate **domain shift and why the scenarios
had to be regenerated**, not as a clean accuracy benchmark for attack
detection.
